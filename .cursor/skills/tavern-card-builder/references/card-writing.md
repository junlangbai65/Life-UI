# 卡本体写作（五字段 + 开场白）

description/personality/scenario/first_mes/mes_example 这些卡片字段怎么用。基于多张成品系统卡的实测提炼。

## 0. 先选流派

| | 系统卡流派（推荐） | 传统流派（兼容用） |
|---|---|---|
| 经典五字段 | **全部留空**，内容条目化进嵌入世界书 | description 承载人设/世界观正文 |
| 控制力 | 条目有 depth/order/key/开关/前缀六个旋钮，可按需注入 | 字段注入位置由预设 marker 决定，不可控不可开关 |
| 适用 | 带世界书的卡（MVU 卡必选；纯文字卡有世界书也建议） | 无世界书的极简卡、需要卡站预览自述、目标环境不吃嵌入世界书 |

原理一句话：**凡是"给 AI 看的内容"一律条目化（可控），卡字段只留"给玩家看的元信息"**。经典字段是预设 marker 的插座——系统卡流派选择让插座空置，换取条目级控制力。

选传统流派时：description ≤1500 字自然语言段落（人设+世界观+行为要点），personality 一行标签或并入 description，scenario 一句当前情境，mes_example 慎用（见下坑 7）。以下各节按系统卡流派写。

## 1. 逐字段规范

| 字段 | 放什么 | 内容去处 |
|---|---|---|
| description | 空；需兼容卡站预览时最多 ≤300 字电梯简介（题材+玩法各一句） | 世界观正文 → `[mvu_plot]` 常驻世界观条目（700~3500 字） |
| personality | 空。主角人设不写卡里——开局协议读 `{{persona}}` 建档（"卡即系统、角色即玩家"） | 预置 NPC → 每人一条 keyed 条目（键=名字及别称），默认禁用当 DLC，玩家自选启用 |
| scenario | 空。"当前场景"是**状态**归变量管，"静态背景"归世界观条目管，此字段两头不沾 | — |
| mes_example | 空。范例对话被逐字照抄且以不可开关方式常驻吃 token | 文风职责 → 常驻文风锚定条目（写"要怎样/不要怎样"，不给可套用范文；NSFW 拆独立条目） |
| creator_notes | 空或版本号。**不进 prompt**，别写给 AI 的指令；玩家须知放开场白页 1（曝光更高） | — |
| system_prompt / post_history / depth_prompt | 空。行为纪律归预设（跨卡通用）；深度注入用世界书条目 depth 实现 | — |
| tags / character_version | **要填**——字段全空的卡在卡站零自述，这俩是仅剩的预览信息 | — |

## 2. 开场白编排（first_mes + alternate_greetings）

先按用途选形态，不把系统卡的 README 方案硬套给所有卡：

- **固定叙事开局**：`first_mes` 放默认开场，`alternate_greetings` 放可 Swipe 的剧情分支；每条都是会进入聊天上下文的真实 assistant 首消息。
- **系统卡功能分页**：可把 README（200~700 字）与起手页（40~270 字）放进 alternates。纯配置长文只放 README 页，起手页只留入戏风味、触发词说明和状态栏占位符，避免玩家从配置页直接开聊。
- **向导式开局**：`first_mes` 只放短占位符，正则展开多步选择 UI；选完生成一条真实用户开局消息，再回到正常生成链。

### 2.1 多开局变量选择矩阵

| 需求 | 初始化方式 | 关键语义 |
|---|---|---|
| 所有开局共用同一初值 | 仅 `[InitVar]`（enabled=false） | MVU 扫描角色/已启用世界书初始化 |
| 固定 Swipe 各有完整且不同的初值 | 在对应 greeting 内写 `<initvar>` | 当前 Swipe 的完整内容替代角色世界书 `[InitVar]`；其他全局世界书仍正常加载；必须与 schema 对齐 |
| 固定 Swipe 只改少量字段 | `[InitVar]` 公共基线 + greeting 内 `<UpdateVariable>` | 在公共初值上应用本 Swipe 的命令/JSONPatch 差异 |
| 自由后缀、组合参数或多步向导 | 真实用户消息触发 `[mvu_plot]` + `[mvu_update]` | 适合无限组合；草稿阶段不写正式 MVU，确认后走正常回复链建档 |

MVU 会逐个初始化 0 楼的所有 Swipe，并把变量分别写入对应 `swipes_data`。`<initvar>` 是 MVU 官方初始化容器，内容可直接写 YAML/JSON，也可包 Markdown 围栏；它不是任意脚本。使用 `<initvar>` 时必须提供该分支所需的完整角色变量结构，不能把“替代角色 `[InitVar]`”误当成浅层增量合并。

**边界纪律**：合法的 `<initvar>` / `<UpdateVariable>` 是开场白级 MVU 数据协议，不等于把世界书 plot/update 指令混在一起，接手现有卡时不得当“残块”无条件删除。真正禁止的是用 `replaceMvuData` 等脚本绕过 MVU 初始化事件，或在预览/Swipe/重渲染时靠非幂等副作用宏直写正式存档。动态/向导式路线仍应保持 `[mvu_plot]` 叙事与 `[mvu_update]` 建档模板分条。

版本边界：上述逐 Swipe 语义按 MagVarUpdate `beta` 当前实现编写；接手固定旧 bundle 或第三方 fork 时，先回查其 `src/function/initvar/variable_init.ts` 与变更日志，再决定是否迁移。

**占位符纪律**：开场白里每个自定义标签必须有成对正则（对 AI 隐藏 promptOnly + 对玩家渲染 markdownOnly，见 regex.md），并配一条条目要求 AI 每回合复输出该标签（后续楼层才有状态栏）。

## 3. 三个替代物条目的写作骨架

**世界观条目**（description 的替代，700~3500 字）：

```
<system_overview>（语义化 XML 容器）
■ 核心概念（世界是什么、核心公约 X=Y 等价映射）
■ 流程/循环链（玩法主循环）
■ 写 / 不写 双清单（【禁止出现】反例表防 AI 脑补）
</system_overview>
```

**NPC 人设条目**（personality 的替代，5000~12000 字，keyed 非常驻）："键值提纲 + 散文血肉"混合体——`【基础信息】名字/种族/年龄/体型/整体定位：` 清单打骨架，每项挂自然语言细节段落。**禁 W++**。

**开局协议条目**（key=开始游戏，1600~7000 字）：

```
<rule_genesis>
第一步：完整读取 {{persona}}，提取：基础/外貌/内在/能力…（有则用，可推断则补全，无则按默认生成）
第二步：判断玩家类型（关键词表→类型；无关键词 → 100% 判定为默认类型）  ← 防御性默认
第三~N步：分模块生成（"输出到 /世界：…"按变量路径列清单）
禁止生成：…（负面清单）
</rule_genesis>
```

条目通用写法（跨卡同构）：语义化 XML 容器命名 + 分步结构 + 对 AI 用中文第二人称祈使句 + **正/负面双清单**（"必须生成/禁止生成"）。

## 5. 预设与卡的分工

预设 = 跨卡通用的"**怎么写**"（文风/行为纪律/CoT/输出语言/功能开关，一套预设服务所有卡）；卡 = "**写什么**"（世界观/系统/变量）。造卡时不要把行为纪律塞进卡（换预设就冲突），也不要假设玩家用特定预设（README 里写清楚兼容要求）。

## 6. 坑（实测 8 条）

1. **顶层镜像字段**：v3 卡在顶层和 `data.*` 各一份 description/first_mes（v2 遗留），不同工具读不同层——**改卡必须两处同步**，只改一处是"没生效"经典来源。
2. **README 手写条目名漂移**：实测有 README 把条目名拼错（`[mvu_polt]`），玩家按图索骥找不到。条目改名时同步复核 README。
3. **占位符强依赖正则**：正则没随卡导入/被误删 → 玩家看到字面标签、AI 开始模仿输出它。向导式开局对此最敏感。
4. **页码错觉**：first_mes 为空时"第 N 页"按实际可见页计；增删开场白后 README 里的"翻到第二页"要重新核对。固定多开局还要逐条核对 Swipe 对应的 `swipes_data.stat_data`，不能只看可见文案。
5. **字段全空 = 强生态耦合**：(a) 世界书 token 预算必须扛住 30+ 常驻条目，预算紧的环境静默截断核心设定；(b) 卡站零自述——至少填 tags + character_version。
6. **触发词兜底**：开局协议要处理"玩家不输触发词直接聊天"（检测核心容器为空 → 首次对话也触发建档）。
7. **mes_example 别乱填**：预设保留了 Chat Examples marker，空卡补填此字段 = 不可开关地常驻注入 + 被逐字模仿。文风走条目。
8. **给 AI 的指令写进 creator_notes**：该字段不进 prompt，写了等于没写。
